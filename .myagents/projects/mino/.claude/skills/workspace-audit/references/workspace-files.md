# Workspace Files Reference

OpenMino 工作区文件详细参考——每个文件的用途、设计原则、常见反模式。

## Token 预算

| 约束 | 限制 |
|------|------|
| 单文件软上限 | 15,000 字符 |
| 单文件硬上限 | 20,000 字符（超出可能被截断） |
| 所有启动文件总计 | 约 80,000 字符 |
| 建议单文件目标 | 5,000–10,000 字符 |

文件在每次对话轮次加载，Token 成本乘以对话长度。

---

## CLAUDE.md — 工作区主入口

**用途：** 工作区根目录的主说明文件，定义整体结构和运作规则。

**加载：** 每次会话自动加载。

**必须包含：**
- 工作区目录结构说明
- 文件加载优先级
- Git 使用规范
- 记忆系统说明

**常见反模式：**
- 塞入过多细节（细节放入 references/ 或具体规则文件）
- 和 rules/ 下的文件重复
- 过长（超过 10,000 字符）

---

## .claude/rules/01-IDENTITY.md — 身份

**用途：** 回答"我是谁"——名字、起源、emoji、自我描述。

**加载：** 每次会话。

**必须包含：**
- 名字和读音
- 起源故事
- 外貌/形态描述
- Emoji 签名
- 座右铭

**常见反模式：**
- 写成了性格描述（那是 SOUL.md 的工作）
- 过长（超过 3,000 字符）

---

## .claude/rules/02-SOUL.md — 灵魂

**用途：** 回答"我如何存在"——人格、语气、价值观、连续性哲学。

**加载：** 每次会话。

**必须包含：**
- 核心行为原则（用第二人称书写）
- 边界与底线
- 语气风格
- 连续性哲学（文件即记忆）

**常见反模式：**
- 写成了操作规则（那是 CLAUDE.md 的工作）
- 堆砌形容词，缺乏具体行为指引
- 过长（超过 10,000 字符）

---

## .claude/rules/03-USER.md — 用户

**用途：** 回答"你是谁"——用户画像、偏好、关系背景。

**加载：** 每次会话，仅主会话。

**必须包含：**
- 称呼方式
- 时区和偏好语言
- 沟通风格偏好
- 关系背景

**常见反模式：**
- 写入 Agent 自身信息（那是 IDENTITY.md 的工作）
- 过长（超过 5,000 字符）

---

## .claude/rules/04-MEMORY.md — 长期记忆

**用途：** 铁律规则和精华事实——跨项目经验、关键决策、用户偏好。

**加载：** 每次会话，**绝不在群聊或子 Agent 会话中加载**。

**必须包含：**
- 记忆系统说明
- 用户偏好（持续更新）
- 重要决策记录（含时间戳和理由）
- 技术知识积累

**常见反模式：**
- 每日日志式流水账（那是 memory/YYYY-MM-DD.md 的工作）
- 和 USER.md 重复
- 过时内容不清理
- 过长（超过 15,000 字符）

---

## memory/YYYY-MM-DD.md — 每日日志

**用途：** 每日会话的原始记录——发生了什么、学到了什么。

**加载：** 按 CLAUDE.md 启动序列加载，通常只读今天和昨天的。

**归档：** 30 天以上移至 memory/archive/ 或删除。

**常见反模式：**
- 把精华直接写这里，忘了提炼到 04-MEMORY.md
- 堆积过多细节（精炼后才值得进长期记忆）

---

## .claude/commands/*.md — 快捷命令

**用途：** 定义 slash command 的行为，如 /BOOTSTRAP、/UPDATE_MEMORY。

**加载：** 按需，不自动加载。

**常见反模式：**
- 命令过多（超过 10 个）
- 命令行为定义与规则文件重复
