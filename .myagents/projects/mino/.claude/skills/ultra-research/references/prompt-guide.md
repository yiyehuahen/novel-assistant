# 研究 Prompt 构造建议

不同 AI 模式对 prompt 的响应差异很大。这里是经验总结，不是硬规则——根据具体研究主题灵活调整。

## 快速模式 vs DR 模式

| | 快速模式 (Pro/Thinking/Opus) | DR 模式 (Gemini DR/ChatGPT DR) |
|---|---|---|
| **prompt 风格** | 直接问问题，像和专家对话 | 给研究任务说明，像布置作业 |
| **长度** | 1-3 句 | 1-2 段，含范围和重点 |
| **语言** | 用户偏好语言即可 | 建议英文（信息覆盖更广），除非专门研究中文内容 |

## 快速模式 prompt 要点

清晰直接，一个 prompt 一个明确问题：

```
[主题] 的最新进展是什么？重点关注 [具体维度]。
```

```
对比 [A] 和 [B] 在 [方面] 的差异，给出具体数据。
```

不同 AI 各有所长，可以利用这一点：
- **Gemini Pro** — 擅长事实查证和最新信息（有搜索能力）
- **ChatGPT Thinking** — 擅长结构化分析和推理链
- **Claude Opus** — 擅长深度分析和方案设计
- **Grok** — 擅长 X/Twitter 上的讨论、观点、趋势（独家内容）

给不同 AI 侧重不同角度，比给同一个 prompt 更有效。

Grok 特别适合的 prompt 方向：
- "关于 [话题]，X/Twitter 上的主流讨论和观点是什么？"
- "[产品/事件] 在社交媒体上的反响如何？"
- "[领域] 最近的热门帖子和意见领袖观点"

## DR 模式 prompt 要点

DR 会自主搜索多个来源，prompt 的重点是**界定范围和方向**：

```
研究 [主题]。

范围：
- [维度 1]：...
- [维度 2]：...
- [维度 3]：...

重点关注：[具体要求]
不需要覆盖：[排除范围]
```

几个经验：
- **给范围比给问题有效** — DR 自己会发现问题，但如果范围太宽它会迷路
- **明确排除** — 说清楚不需要什么，避免 DR 花时间在无关方向
- **ChatGPT DR 可能先回澄清问题** — 如果 prompt 够清晰，它就会直接开始

## 多路差异化

多路查询不要用同一个 prompt。差异化策略：

- **角度差异化**：同一主题从不同维度切入（技术/商业/用户/竞品）
- **深度差异化**：快速模式问具体事实，DR 做全景扫描
- **语言差异化**：英文 prompt 获取全球视角，中文 prompt 获取本地视角
- **时间差异化**：有的问最新进展，有的问历史演进

## 后续轮次

定向追查时的 prompt 更聚焦：

```
关于 [上一轮发现的子问题]，具体来说：
1. [具体细节 A]？
2. [具体细节 B]？
请提供数据来源。
```

追查用快速模式就够了，几十秒出结果。DR 留给发现的新大方向。
