# 2026-04-05

## OpenHarness Hindsight MCP 修复

### 问题
OpenHarness 连接 Hindsight MCP 时报错：
```
RuntimeError: Attempted to exit a cancel scope that isn't the current task's current cancel scope
```

### 根因
anyio 的 `create_task_group()` 与 `Process` 资源管理在 Windows 上有 cancel scope bug。

### 修复方案

**1. stdio_client.py 核心修复**
- 文件：`OpenHarness/.venv/Lib/site-packages/mcp/client/stdio/__init__.py`
- 使用 `asyncio.create_subprocess_exec` 替代 anyio Process
- 移除 `async with (tg, process,)` 资源管理模式
- 任务函数内捕获所有异常防止 task group 损坏

**2. mcp/client.py 错误抑制**
- 文件：`openharness/src/openharness/mcp/client.py`
- 在 `close()` 中抑制 cancel scope RuntimeError

**3. hindsight-stdio.py 响应格式**
- 文件：`~/.openharness/hindsight-stdio.py`
- `tools/call` 结果必须包装为 `CallToolResult` 格式
