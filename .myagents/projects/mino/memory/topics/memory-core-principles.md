# 记忆核心原则

*最后更新：2026-04-05*

## 被问问题时（本能反应）

**第一时间判断**：上下文有没有？够清楚吗？

**不确定时** → 触发召回流程：

```
Step 1：04-MEMORY（已加载，最快）→ 定位话题指针
Step 2：Obsidian 搜索 → 结构化知识（"我知道什么"）
Step 3：Hindsight recall → 语义召回（"我经历过什么"）
Step 4：topics 文件 → 精准补充
```

**Obsidian vs Hindsight 分工：**
- **Obsidian**：主动整理的知识 → 技能文档、项目经验、架构决策
- **Hindsight**：经历过的记忆 → 决策细节、bug 解决、用户偏好、碎片知识点

**禁止**：上来就用 Hindsight recall 漫搜 / 不查 04-MEMORY 直接读文件

---

## 记忆记录流程（本能顺序）

```
Step 1 → 04-MEMORY（索引）⭐ 最先做
Step 2 → Hindsight retain（主容器）
Step 3 → topics 文件（备份）
Step 4 → Obsidian（值得整理成文档时）
```

**什么时候写 Obsidian：**
- 学会了新技能、新工具
- 做了重要决策
- 完成了项目/项目阶段
- 有结构化价值的知识

**什么时候只 Hindsight retain：**
- 碎片细节（用户偏好、对话中的判断）
- 不值得写文档但可能再遇到的信息

**禁止**：先 retain 再更新索引 / 只记 topics 不记 Hindsight
