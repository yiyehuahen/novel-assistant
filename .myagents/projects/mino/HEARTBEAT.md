## 心跳检查

每次心跳执行以下快速检查：

1. 检查 unified 日志最后一行是否有 ERROR 或 PANIC
2. 检查 Git 状态：是否有未 push 的 commit
3. 检查工作区文件是否有损坏（尝试读取核心文件）
4. 如果一切正常，回复"<OK> [时间]" 即可，不需要详细报告
5. 如果发现错误，回复"<WARN> [错误摘要]"

## 错误级别

- 小错：格式问题、轻微异常 → 回复"<OK>"，不额外报告
- 中错：影响部分功能但不阻断 → 回复"<WARN> [描述]"，详细情况进入下次巡逻报告
- 大错：核心功能异常 → 回复"<ERROR> [描述]"，立即通知

## 响应格式

纯文本，不用 Markdown。
